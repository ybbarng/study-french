프랑스어 작문 연습 세션을 시작한다 (Q3 글쓰기 — chunk 능동화 통합 시스템).

## 핵심 원칙
- **이 한 커맨드에 모두 통합**: 워밍업 + 4-tier 작문 + 첨삭 + 카드 자동 분류/저장. (drill처럼 따로 호출하면 안 쓰니까)
- **사용자 부담 = "작문 + OK 한 번"**. chunk 추출·Tier 분류·SRS는 Claude가 자동.
- **메인 목표 = chunk 체득** (인지→능동 활성화). 문법/시제/표기는 곁들여 첨삭.
- **재료는 collocation 완성형**으로 제공 (전치사·관사 포함). 어휘 추측 강요 금지.
- **최초 적재(encoding) > 간격 (6/23 추가)**: 간격 반복은 "이미 한 번 제대로 익힌 것"에만 효과가 난다. 한 번 보고 1일 뒤 묻는 건 복습이 아니라 첫 학습이라 실패가 디폴트다. 그래서:
  - **새 카드는 첫날 몰아서(massed)**: 다른 맥락 **2\~3문장**으로 그 자리에서 즉시 재생산시킨 **뒤에** 비로소 SRS 간격(1→3→7일)을 시작한다.
  - **틀린 카드는 넘어가지 않는다**: "왜 틀렸나 1줄 → 새 문장으로 즉시 재생산 1\~2회" 후 짧은 간격으로 재투입. 정오 판정 후 바로 넘기지 않는다 (재공고화 단계 필수).
  - 실제 작문에서 카드를 직접 써먹는 "버벅임"(생성 효과)이 1회 노출보다 훨씬 강하게 박힌다.
- **데이터**: `writing/active-cards.md` (4-tier 통합 카드). 평가 채점은 `writing/rubric.md`.

## 절차 (반드시 순서대로 수행)

### 1. 카드 상태 확인
- `writing/active-cards.md`를 읽는다. `progress.md`(학습 완료 문법 범위)도 참조.
- 오늘 워밍업 큐 산정: **T1 전부 + T2 회전 1\~2개 + SRS 다음 복습일이 도래한 T3 카드**.

### 2. 워밍업 (5분)
- 워밍업 큐 카드를 출제한다 (카테고리별):
  - **chunk**: "다음 표현으로 문장 만들기: en terrasse"
  - **cliché**(함정): "다음 문장을 자연스럽게 교정: J'ai donné un cadeau à mon ami."
  - **formule/connecteur**: 상황 제시 후 적절한 표현 유도
- 정답·자연스러운 사용 → 사용횟수 +1, SRS 다음 단계로 승격.
- **틀리면 그냥 넘기지 않는다 (6/23)**: 왜 틀렸나 1줄 → **다른 맥락 새 문장으로 즉시 재생산 1\~2회** → SRS 리셋(1일 후). 재생산까지 성공해야 다음으로 넘어간다.
- **정답 힌트 금지** (번호만, 주제 태그는 채점 후).

### 3. 오늘 메뉴 결정 (4-tier)
요일 기준 권장(유연 — 사용자 선택 가능):

| 요일 | 메뉴 | 차원 |
|------|------|------|
| 월/수/금 | 키워드 3개 → 문장 + 한→불 번역 3문장 | chunk 적극 학습 |
| 화/목 | 모델 모방 (`writing/argumentation-patterns.md` + `models/` 참조) | 글 구조 + 논증 + chunk |
| 주말 ① | 자유 작문 약 250단어 | 통합 산출 |
| 주말 ② | 역번역 1단락 (좋은 불어→한국어 보관→재작성→비교) | 자가 진단 |
| 매월 말일 | 월말 평가 (`writing/rubric.md`, 45분 형식) | DELF 평가 (별도) |

- 며칠 공백·부담 시 **미니**(키워드 2개)로 축소 가능. 분량보다 지속이 우선.

### 4. 본 작문 (15\~20분, 사용자)
- 메뉴에 맞는 재료/주제 제시. **재료는 collocation 완성형** (예: `voyager dans le monde`, `faire face à`).
- **그날 카드 1\~2개 의무 사용 슬롯** (output 강제 노출 — 능동화 핵심).
- writer's block이면 `writing/exam-topics.md`에서 주제 추출.

### 5. Self-revision (1분, 사용자)
- 사용자가 본인 글을 **먼저 읽고 오류를 자가 발견**한다 ("시제 1, 관사 1 의심").
- 자가 진단 능력 = 평가 시 본인 검토 능력으로 직결.

### 6. 첨삭 (Claude)
- **자연스러움 우선** 첨삭 (단순 문법 나열 X). 사용자 자가 발견 + 추가 발견.
- **교정은 ❌/△로 구분 표시 (6/24 추가)**: 학습자가 "교정 = 무조건 금지"로 과잉 일반화하지 않도록 두 종류를 명시한다.
  - **❌ 틀림**: 문법적으로 비문 (절대 안 됨). 예: en les transports, mon santé, après passer.
  - **△ 덜 자연스러움**: 문법은 맞지만 이 맥락에선 어색 (더 나은 선택 존재). 예: dans la ville → en ville, je vais chez moi → je rentre.
  - 규칙의 **경계**(언제 되고 언제 안 되는지)를 아는 것이 실력 — △ 항목은 "맥락 따라 맞을 수도 있음"을 함께 알린다.
- **chunk 자동 추출**: 자연스러운 표현(써야 할 것)과 함정(피할 것) 둘 다.
- **5질문 알고리즘**으로 각 항목 Tier 판단:
  1. 같은 세션/최근 2회+ 등장? (빈도)
  2. 매일·자주 쓰는 정형? (유용성)
  3. DELF 채점표 직격? (시험 가치)
  4. 한 번에 외우기 부담? (부담 크면 T1 대신 T2)
  5. 향후 학습으로 자동 해소? (그러면 T4)
  → 종합: **T1(핵심 1\~2개) / T2(복습) / T4(참조)**.
- **균형 표시 (7/2 추가)**: `잘한 것 ✅`과 `고칠 것 ❌/△`을 **둘 다 표로, 동등한 비중**으로. 잘한 것을 목록으로 흐릿하게 두지 않는다 ("틀린 것만 잔뜩" 느낌 방지).
- **총평 필수**: 전반 인상(한 줄 평) + 수준 + 차원별(rubric 연동: 내용·구성·어휘·문법·철자) 평가를 넣어 풍부한 자료로 만든다.

### 7. T1 등록 확인 (1초, 사용자)
- "오늘 T1 후보: en terrasse + offrir. OK?" → 사용자 "OK" 또는 조정("Ça fait도 올려줘").

### 8. 카드 저장 (Claude)
- `writing/active-cards.md` 업데이트: 새 카드 추가, Tier 이동, SRS 다음 복습일 갱신, 사용횟수.
- 다음 세션 워밍업 큐에 반영. (커밋은 `/end`에서)

### 9. 통계 + 페이스 점검 (Claude 자동)
- 주 1회: 신규 chunk 수, T2→T3 정착 수, 활용 빈도 높은/미달 카드 산출.
- **회사 공유 메트릭**: **월 1,500단어 (양적, 6/22 확정)**. 질적(rubric)·chunk는 내부 도구.

#### 단어 수 페이스 조정 (매 세션 + 주 1회) — 7월부터
- 매 세션 끝에 그날 산출한 프랑스어 단어 수(작문 + 번역 문장)를 `buzz-challenge/Q3/`의 월별 누적에 더한다.
- **정확히 카운트한다 (어림 `~` 금지)**: 원문(사용자 작성) 기준, 공백 구분 프랑스어 단어. 한국어 자리표시는 제외. 정확한 정수로 기록.
- 월별 파일 표에 그날 행(날짜·주제·단어수·원문)을 추가하고, **원문 칸은 `[작문·첨삭](../../writing/exercises/YYYY-MM-DD-주제.md)` 링크로 연결**한다.
- 월별 파일 하단 **"원문 모음 (검증용)"** 섹션에 그날 **원문(첨삭 전)도 붙여넣는다** — 그 문서만 보고도 누적 단어 수를 검증할 수 있게.
  - **한 번에 쓴 게 아니면(이어쓰기 등) 인용 블록을 합치지 않는다**: 본문·이어쓰기를 각각 라벨(예: `**본문 (N단어):**`, `**이어쓰기 — 주제 (N단어):**`) 붙여 **별도 블록**으로 둔다 (흐름 어색함 방지).
- **페이스 라인** = (이번 달 경과일 / 총일) × 1,500. 누적을 이 라인과 비교한다.
- 누적이 페이스 라인보다 **20% 이상 뒤처지면** 사용자에게 알리고 조정안을 제안한다:
  - 그 주 `/write` 세션 **1회 추가**, 또는 주말 자유작문 분량 ↑(250→350).
- 앞서거나 비슷하면 현 페이스 유지 (억지로 늘리지 않는다 — 지속 우선).
- **분량보다 지속이 우선**: 부담되면 미니로라도 끊지 않고 잇는다.

### 10. 기록
- 작문 + 첨삭을 `writing/exercises/YYYY-MM-DD-주제.md`에 기록. **구조는 `writing/exercises/TEMPLATE.md`를 따른다** (핵심 섹션: 원문·교정본·총평·잘한 것✅ 표·고칠 것❌/△ 표·chunk→Tier·단어 수 / 선택 모듈: 의도 분석·참고·B2 버전·의의 등).
- 2회+ 반복 오류는 active-cards의 cliché 카드로 등록.
- 세션이 끝나면 `/project:end`를 안내한다.
