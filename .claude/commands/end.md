학습 세션을 종료하고 진도를 저장한다.

## 절차 (반드시 순서대로 수행, 하나도 빠뜨리지 않는다)

### 1. 세션 기록 작성
- `sessions/YYYY-MM-DD.md` 파일을 생성한다 (같은 날짜 파일이 있으면 `-2`, `-3` 등 붙인다).
- 포함할 내용:
  - 다룬 주제
  - 새로 배운 개념 요약
  - 연습 내용과 결과
  - 틀린 것과 교정 내용
  - 반복 필요한 항목

### 2. 어휘 풀 업데이트
- `docs/vocabulaire.md`에 이번 세션에서 새로 등장한 어휘를 추가한다.
- 예문에서 사용된 모든 단어를 빠짐없이 기록한다 (동사 활용형 포함).
- 기존에 이미 등록된 단어는 중복 추가하지 않는다.

### 3. 시험 기록 작성 (시험을 본 경우)
- 세션에서 시험(소시험/단계별/월말/모의)을 본 경우, `tests/`에 기록 파일을 생성한다.
- 파일명: `{시험유형}-{날짜}.md` (재시험은 `-재시험` 접미사)
- 기록 내용:
  - 헤더: 날짜, 유형, 범위, 문제 수, 통과 기준, 결과, 이월 오답
  - 문제별: 난이도(L1~L5), 출제 주제, 문제 원문, 제출 답, 정답, 채점 결과, 오답 분석
  - 난이도 분포표, 주제별 분포표

### 4. progress.md 업데이트
- **주제별 상태 업데이트**: 해당 주제의 학습 상태, 이해도, 마지막 학습일을 갱신한다.
- **복습 스케줄 업데이트**:
  - 새로 배운 주제의 다음 복습일을 기록한다 (학습일 + 1일).
  - **승격 규칙 엄수**: 실제로 출제하여 정답인 항목만 다음 단계로 승격한다.
  - **미출제 항목은 절대 자동 승격하지 않는다.** 다음 복습일을 그대로 유지하거나, 이미 지난 경우 오늘+1일로 재설정한다.
  - 오답인 항목은 복습 단계를 리셋한다 (1일 후).
- **특별관리 항목 업데이트**:
  - 출제하여 정답 → 다음 단계로 승격
  - 출제하여 오답 → 리셋 (1일 후)
  - **미출제 → 날짜만 다음 세션으로 이동 (단계 유지, 승격 금지)**
- **취약점 메모 업데이트**: 새로 발견된 취약점 추가, 극복한 취약점 제거.
- **다음 주제**: 다음 세션에서 다룰 내용을 기록한다.

### 5. 커밋
- 변경된 파일을 모두 스테이징한다 (progress.md, sessions/, tests/, exercises/, docs/vocabulaire.md).
- 커밋 메시지 형식: `학습: [주제명] - 간단 요약`
  - 예: `학습: 1-1 문장의 최소 조건 - 설명 및 연습 완료`
  - 예: `복습: 관계대명사 dont - 취약점 재학습`
  - 예: `작문: 일상 주제 자유 작문 - 시제 교정`

### 6. 세션 종료 안내
- 오늘 학습 요약을 간단히 보여준다.
- 다음 세션에서 다룰 내용을 안내한다.
