이전 학습에서 취약했던 부분을 복습한다.

## 절차 (반드시 순서대로 수행)

### 1. 복습 부채 전체 파악
- `progress.md`의 "복습 스케줄" 섹션을 읽는다.
- 오늘 날짜 기준으로 **복습 예정일이 지난(overdue) 항목**을 전부 나열한다.
- `progress.md`의 "특별 관리 항목" 섹션에서 예정일이 지난 항목도 나열한다.
- `progress.md`의 "취약점 메모 (활성)" 섹션에서 항목을 수집한다.
- 최근 세션 기록(`sessions/`)에서 틀린 내용을 수집한다.
- 이해도가 "하" 또는 "중"인 항목을 찾는다.
- **밀린 항목 전체 수와 목록**을 사용자에게 보고한다.

### 2. 복습 대상 우선순위 결정
- 우선순위:
  1. 가장 오래 밀린(overdue가 긴) 항목
  2. 반복적으로 틀리는 패턴 (취약점에 여러 번 등록된 것)
  3. 이해도 🔴(하) 항목
  4. 이해도 🟡(중) 항목
  5. 특별관리 항목 중 미출제로 넘어간 것
- 한 세션에서 처리할 수 있는 분량을 현실적으로 산정한다 (15~25문제).

### 3. 복습 진행
- 해당 개념을 다시 간략히 설명한다 (필요한 경우).
- 이전에 틀렸던 것과 유사한 연습을 제시한다.
- **한 문제씩** 출제한다.
- 결과에 따라:
  - 정답 → 복습 스케줄에서 다음 단계로 승격
  - 오답 → 복습 단계를 리셋 (1일 후)하고 취약점에 유지
- **미출제 항목은 절대 자동 승격하지 않는다.**

### 4. 세션 중 기록
- 복습 결과를 메모해둔다 (항목별 정답/오답).
- 세션이 끝나면 `/project:end`를 안내한다.
