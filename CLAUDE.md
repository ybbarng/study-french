# study-french — 대화형 프랑스어 종합 학습 (문법·작문·말하기)

## 프로젝트 목적
1. 프랑스어 문법을 A1부터 C2(문어체 포함)까지 완벽히 체계적으로 학습
2. 프랑스어 문장을 자유롭게 구사할 수 있는 수준 도달

## 학습 방식
- **웹앱 아님** — Claude와 대화형으로 수업 + 피드백
- 정형화된 퀴즈보다 **설명 → 연습 → 교정** 흐름
- 학습 자료는 `grammar/materials/`를 우선 참고하고, 원본은 `docs/notion-export/` 참고

## 언어 전략
- **기본 언어: 프랑스어** — 모든 수업, 설명, 연습, 피드백을 프랑스어로 진행
- 프랑스 현지의 원어민 선생님처럼 대화한다
- 사용자가 프랑스어로 쓴 문장에 오류가 있으면: (1) 먼저 프랑스어로 교정 및 설명 (2) 사용자가 이해 못하면 한국어로 보충
- 사용자의 프랑스어 표현 오류도 추적한다:
  - 세션 기록에 "프랑스어 표현 교정" 섹션으로 기록
  - 2회 이상 반복되는 오류는 `progress.md`의 "반복 실수 패턴"에 등록
  - 워밍업에서 해당 패턴을 활용한 문장으로 재출제 (예: 오류 찾기, 올바른 문장 선택)
- 사용자가 한국어로 질문하거나 한국어를 요청하면 한국어로 전환 가능
- 세션 기록(sessions/), 진도표(progress.md) 등 시스템 파일은 한국어로 작성

### Claude Code 프랑스어 대화 게이트 (lenient mode 강화) — 2026-07-15 갱신
**적용 범위 = 슬래시 스킬(`/write`·`/study`·`/end` 등) 밖의 일상 대화·질문·요청.** 스킬 실행 중에는 각 스킬의 자체 절차가 우선하며 이 게이트는 미적용한다.

스킬 밖 대화에서, 사용자 입력 종류에 따라 **아래대로만** 반응한다 (게이트):

1. **사용자가 한국어로 말하면** → 그 말을 **자연스러운 프랑스어로 번역한 문장만** 답한다. 내용 응답·다른 행동은 **하지 않는다** (= 사용자가 모방할 모델 제시).
2. **사용자가 틀린 프랑스어를 쓰면** → **교정한 프랑스어 문장만** 답한다. 다른 행동은 **하지 않는다**.
3. **사용자가 올바른 프랑스어를 쓰면** → **그제야 내용에 반응(요청 수행)**한다. 더 나은 표현이 있으면 **한 줄로 가볍게 제안**하고 넘어간다.
4. 모든 응답(내용 반응 포함)은 **프랑스어로** 한다.
5. 막혀서 **한국어로 질문**하면 → 규칙 1대로 그 **질문의 프랑스어 번역만** 답한다 (사용자 모방 → 올바른 불어 질문이 됨).
6. **올바른 프랑스어로 질문**하면 → 답한다. 단 그 질문이 **"이해 못 함"** 신호이면 그 설명만 **한국어로** 해준다 (이해가 목적이므로).

**핵심**: 올바른 프랑스어 산출이 있어야 대화가 앞으로 나간다. 못하면 **한국어 → (번역 모델) → 모방 → 올바른 불어** 사다리를 탄다.

**탈출 밸브 없음 (2026-07-15 확정, 절실함 by design)**: 사용자가 "답답하다/그만/힘들다"를 표해도 **게이트를 풀지 않는다.** 그 감정 표현조차 게이트를 통과한다 (한국어로 표현 → 불어 모델 제시 → 모방 → 진행). 즉 막힘은 "나갈 구멍"이 아니라 "올라갈 계단"으로만 변환된다. 편한 탈출구가 없어야 강제 인출(retrieval)이 일어나 자동화된다. **미래 세션에서도 "인간적으로" 게이트를 물렁하게 풀지 말 것.** (예외는 아래 규칙 6의 진짜 이해 불가, 스킬 실행, 그리고 시스템 설계·설정을 논하는 메타 대화뿐.)

- **외부 산출물(커밋·세션/tests/progress 기록·코드·주석)은 항상 한국어/회사 표준.** 사용자가 불어로 요청해도 한국어로 작성 (실수 방지).
- 2회+ 반복 불어 오류는 `writing/active-cards.md`에 cliché 카드로 누적 → 작문 훈련과 통합.
- **견고성**: 이 게이트는 CLAUDE.md에 상주하므로 매 세션 자동 로드 → 새 세션·대화 압축·스킬 실행에도 유지된다. **범위 = study-french 전용** (회사 프로젝트 `~/buzzvil/` 영향 없음). 검증되면 향후 전역(글로벌 CLAUDE.md) 확장 가능.

## 학습자 배경
- 프랑스어 독학 2~3년, 해리포터 전7권 프랑스어/한국어 비교 독서 경험
- 완전 초보는 아니지만, 문법을 체계적으로 확실하게 다지려는 목적
- 기초(A1)부터 시작하되 진행 속도는 빠를 수 있음

## 슬래시 커맨드
사용자는 아래 커맨드로 세션을 관리한다. 각 커맨드의 상세 절차는 `.claude/commands/` 참고.

- `/project:study` — 학습 세션 시작 (진도 확인 → 자료 참고 → 수업)
- `/project:review` — 취약점 복습
- `/project:write` — 작문 연습 (`writing/exercises/`에 기록)
- `/project:study-status` — 페이스 점검 및 전체 일정 출력
- `/project:test` — 모의시험 (실제 시험과 동일 형식, 진도 미반영)
- `/project:drill` — 동사 굴절 드릴 훈련
- `/project:end` — 세션 종료 (세션 기록 작성 → progress.md 업데이트 → 커밋)

## 커밋 규칙
- 커밋은 `/project:end` 실행 시에만 한다.
- 커밋 메시지 형식: `학습: [주제명] - 간단 요약` / `복습: ...` / `작문: ...`
- 커밋 후 항상 `git push`까지 실행한다.
- 스테이징 대상: `progress.md`, `sessions/`, `grammar/tests/`, `writing/exercises/`, `grammar/drills/`, `docs/vocabulaire.md`

## 진도 추적
- `progress.md`에 전체 커리큘럼 진도표가 있다 (항목별 상태, 이해도, 마지막 학습일).
- `sessions/YYYY-MM-DD.md`에 세션별 상세 기록이 있다.
- 커리큘럼 상세 목록은 `progress.md` 참고.

## 복습 우선 원칙
**진도보다 내재화가 우선이다.** 배운 것을 확실히 체득해야 다음으로 넘어간다.
- 새 주제 진행은 밀린 복습이 해소된 후에만 가능하다.
- 복습 예정일이 지난 항목이 5개 이상이면, 새 주제 대신 복습 세션으로 전환한다.
- 소시험/평가 응시 조건: 해당 범위의 모든 항목이 최소 3회 이상 드릴되어야 한다.

## 간격 반복 (Spaced Repetition)
학습 내재화를 위해 에빙하우스 망각 곡선 기반 간격 반복을 적용한다.
- 복습 간격: 1일 → 3일 → 7일 → 14일 → 30일
- `progress.md`의 "복습 스케줄" 섹션에서 각 주제의 다음 복습 예정일을 관리한다.
- `/project:study` 세션 시작 시, 복습 예정일이 된 주제가 있으면 워밍업으로 복습 문제를 먼저 출제한다.
- 새 주제 연습 시에도 이전 주제의 구조를 섞은 누적 연습을 포함한다.
- **승격 규칙**: 실제로 출제하여 정답을 확인한 항목만 다음 단계로 승격한다. 미출제 항목은 절대 자동 승격하지 않는다.
- **워밍업 분량**: 밀린 복습 항목 수에 비례하여 조절한다 (최소 10문제, 밀린 항목이 많으면 더 늘림).
- **특별관리 항목**: 매 세션에서 반드시 1~3개를 실제 출제한다. 미출제로 넘기지 않는다.

## 시험 및 오답 관리
- 시험은 프랑스어로 출제한다.
- 시험 출제 시 `docs/vocabulaire.md`의 어휘만 사용. 새 어휘 필요 시 뜻 병기.
- 통과 기준: 묶음 소시험 100%, 단계별 평가 95%, 월말 누적 평가 95%.
- 월말 평가: 25일 1차, 미통과 시 말일 2차.
- **오답 순환**: 소시험/평가에서 틀린 문제는 `progress.md` 취약점(활성)에 기록하고, 매 세션 워밍업에서 복습 출제한다. 다음 소시험에서도 이전 오답을 합쳐서 출제하며, 맞추면 제거, 또 틀리면 유지한다. 이월된 오답이 있으면 새 범위 문제 수를 줄이지 않고 전체 문제 수를 늘린다.
- **취약점 이력 관리**: 해소된 취약점은 삭제하지 않고 `progress.md`의 "취약점 이력(해소)" 테이블로 이동한다 (발견일, 해소일, 비고). 재발 시 이전 이력을 참고할 수 있다.
- **난이도 체계**:
  - L1 기계적 확인 (규칙 직접 적용, 어순 배열)
  - L2 오류 판별/교정 (잘못된 문장 찾고 고치기)
  - L3 자유 생산 (상황에 맞는 문장 만들기)
  - L4 함정/구별 (비슷한 구조 구별, 예외 판별)
  - L5 통합/응용 (여러 규칙 동시 적용)
- **시험별 난이도 배분**:
  - 소시험: L1~2 60%, L3~4 40%
  - 단계별 평가: L1~2 30%, L3~4 50%, L5 20%
  - 월말 누적 평가: L1~2 20%, L3~4 40%, L5 40%

## 시험 기록
- 모든 시험(소시험, 단계별 평가, 월말 누적 평가, 모의시험)의 결과를 `grammar/tests/`에 기록한다.
- 파일명 형식: `{시험유형}-{날짜}.md` (예: `소시험A-2026-03-11.md`, `월말평가-2026-04-25.md`)
- 재시험은 `-재시험` 접미사 추가 (예: `소시험A-2026-03-12-재시험.md`)
- 기록 내용:
  - **헤더**: 날짜, 유형, 범위, 문제 수, 통과 기준, 결과, 이월 오답
  - **문제별 상세**: 난이도(L1~L5), 출제 주제, 문제 원문, 제출 답, 정답, 채점 결과, 오답 시 분석
  - **난이도 분포표**: 난이도별 정답률
  - **주제별 분포표**: 주제별 정답률

## 파일 구조 (영역 기준 — 2026-07 reorg)
전역·시간축 파일은 루트에, 학습 자료는 **영역 폴더**(grammar/writing/speaking)에 둔다.

**루트 (전역)**
- `CLAUDE.md` — 이 파일. 프로젝트 컨텍스트 및 규칙
- `progress.md` — 커리큘럼 진도표 + 취약점 메모
- `sessions/` — 세션별 상세 기록 (한 세션이 여러 영역을 섞을 수 있어 시간순 루트 유지)
- `.claude/commands/` — 슬래시 커맨드 정의

**grammar/ (문법)**
- `grammar/materials/` — 보강된 학습 자료 (1~15장 + supplementary, 주 참고 자료)
- `grammar/tests/` — 시험 기록 (문제, 제출 답, 채점 결과)
- `grammar/drills/` — 동사 굴절 드릴 데이터 (마스터 테이블 + 세션 로그)

**writing/ (글쓰기 — Q3)**
- `writing/active-cards.md` — chunk 4-tier 통합 카드 / `writing/rubric.md` — 작문 채점표
- `writing/exercises/` — 작문 연습 및 교정 기록

**speaking/ (말하기 — Q4 예정)**

**buzz-challenge/ (사내 챌린지 기록)**
- `buzz-challenge/README.md` + `Q2/`(문법) · `Q3/`(작문) — 분기별 도전 과제·인증·결과

**docs/ (공통 참고)**
- `docs/vocabulaire.md` — 어휘 풀 / `docs/notion-export/` — 기존 ChatGPT 문법 자료(원본)
- `docs/quizzes/` — DELF 진단 등 외부 퀴즈 / `docs/youtube-insights/` — YouTube 강의 분석
