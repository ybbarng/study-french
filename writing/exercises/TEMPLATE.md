# 작문 exercise 문서 표준 구조

모든 `writing/exercises/YYYY-MM-DD-주제.md`는 이 구조를 따른다.
**목적**: 틀린 것/잘한 것을 **균형** 있게(둘 다 표로) + **총평**으로 풍부한 자료 + 문서 간 일관성.

## 핵심 원칙
- **균형 표시**: `잘한 것 ✅`과 `고칠 것 ❌/△`을 **둘 다 표로, 동등한 비중**으로. 잘한 것을 목록으로 흐릿하게 두지 않는다.
- **총평 필수**: 매 작문에 전반 인상·수준·차원별 평가를 넣어 "틀린 것만 잔뜩" 느낌을 없앤다.
- **이탤릭 생략** (사용자가 입력 어려움). 고유명사는 억지 수정 X, "참고"로만.

---

## 핵심 섹션 (항상)

```
# YYYY-MM-DD 작문 — {주제} ({N}단어)

> {요일 메뉴 유형} · 의무 요소: {카드/구문} · 단어 수 집계: 예/아니오

## 원문 (사용자)
> {원문 그대로 (한국어 자리표시 포함)}

## 교정본
> {자연스러운 교정본}

## 총평
- **한 줄 평**: {강점과 최대 약점을 한 문장에}
- **수준**: {대략 DELF 레벨/방향}
- **차원별** (rubric 연동): 내용·과제수행 {…} / 구성·응집 {…} / 어휘 {…} / 문법·시제 {…} / 철자·표기 {…}

## 잘한 것 ✅
| ✅ | 항목 | 설명 |
|----|------|------|
| ✅ | {강점} | {왜 좋은지 / 어떤 카드·규칙 정착인지} |

## 고칠 것 (❌ 틀림 / △ 덜 자연스러움)
| 표시 | 원문 | 교정 | 설명 |
|------|------|------|------|
| ❌ | {원문} | {교정} | {규칙} |
| △ | {원문} | {교정} | {맥락 따라 맞을 수도 있음} |

## chunk → Tier
- 🔴 T1(1~2): {…}  🟡 T2: {…}  ⚪ T4: {…}

## 단어 수
- {N}단어 → {월} 누적 {누적}/1,500 ({페이스 코멘트})
```

## 선택 모듈 (해당할 때만 추가)

- **## 의도 분석** — 사용자가 표현하려던 의미 (뜻 어긋남이 있을 때, 의미 단위 피드백)
- **## 참고 (고유명사·문화 표기)** — 브랜드=공식 라틴 표기 / 작품=국제 통용 제목 + 카테고리 gloss. 억지 수정 X, 참고만.
- **## B2 격식 버전** — 같은 내용의 시험 톤(soutenu) 교정 (레지스터 확장 학습 시)
- **## 재사용 목표 표현** — 능동화로 콕 집어 반복할 표현 (chunk와 별도 강조)
- **## 모델 (scaffolding)** — 한국어 입력 → 불어 모델 병기 (대화형/막힘 시)
- **## 의의 / 메타** — 이 세션이 보여준 특별한 진전 (예: 워밍업 즉시 전이, 자가 규칙 도출)

## 문서 유형별 권장 모듈
- **키워드/번역 작문** (월수금): 핵심만.
- **자유 작문** (주말): 총평 + 의도 분석 + (선택) B2 버전.
- **베이스라인/월말 평가**: 총평에 **rubric 점수(차원별)** 명시 + 강점/약점.
- **대화 기록**: 모델(scaffolding) + 의의/메타 중심 (교정표는 간략).
